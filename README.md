## ğŸ¯ Usage

### Making a Payment

1. Navigate to the home page
2. Click on "Make Payment" or go to `/payment`
3. Fill in the payment form:
   - **Cardholder Name**: Full name on the card
   - **Card Number**: 13-19 digit card number
   - **Expiry Date**: Month and year (must be future date)
   - **CVV**: 3 or 4 digit security code
   - **Amount**: Payment amount (must be > 0)
4. Submit the form to process the payment
5. View the success page with transaction details

### Viewing Transactions

- Navigate to `/transactions` to view all processed payments
- Each transaction displays:
  - Transaction ID
  - Cardholder name
  - Last 4 digits of card
  - Amount
  - Status
  - Timestamp

## âš™ï¸ Configuration

### Environment Variables

You can customize the following settings in `app.py`:

- `app.secret_key`: Session secret key (auto-generated by default)
- `host`: Server host (default: 0.0.0.0)
- `port`: Server port (default: 5000)
- `debug`: Debug mode (default: True for development)

### Production Deployment

**Important**: Before deploying to production:

1. Set `debug=False` in `app.py`
2. Use a production-grade WSGI server (e.g., Gunicorn)
3. Implement a proper database instead of in-memory storage
4. Add HTTPS support
5. Integrate with real payment processors (Stripe, PayPal, etc.)
6. Implement proper logging and monitoring
7. Add rate limiting and fraud detection
